## 一、项目背景与目标

### 1.1 现状痛点
- 全国青年人才驿站（应届毕业生免费短期住宿）信息**分散在各市人才网/人社局/学校就业办**
- 应届生需要去十几个网站才能拼凑出"我能申请哪些、怎么申请、要什么材料"
- 各地驿站码、政策补贴、申请流程**没有统一入口**

### 1.2 项目目标
为应届毕业生（尤其是粤港澳大湾区）提供一站式查询平台：
- **找驿站** ✅
- **看政策** ✅
- **问 AI** ✅
- **一键申请** ✅
- 后台对接学校就业办分发驿站码，控制访问

### 1.3 关键指标（KPI）
| 指标 | 目标 |
|------|------|
| 收录驿站数 | 珠三角 ≥ 180 ✅，全年扩展至 ≥ 500 |
| 单日 PV | ≥ 1000 |
| 详情页 → 申请按钮转化率 | ≥ 25% |
| AI 助手有效解答率 | ≥ 90% |

### 2.1 主要用户
- **应届/在读毕业生**（本科及以上为主）

## 四、核心功能详细设计

### 4.1 驿站码鉴权系统
- 16 位字母+数字组合（去除易混淆字符 0/O/1/I）
- 状态：`enabled/disabled`，可设过期时间
- 每码统计 `used_count` + `last_used_at`
- 后台批量生成 N 个码后下载 CSV 给学校老师分发

### 4.2 驿站数据
**字段**（部分）：
- 名称、城市、区域、地址、经纬度
- 富文本入住指引（Quill 编辑器）
- 一键申请：联系人、电话、申请链接、微信公众号二维码
- 申请条件 / 所需材料（JSON 数组）
- 免费入住天数

**地理编码策略**：
1. 高德 `/v3/geocode/geo` 标准编码
2. 失败回退到 `/v3/place/text` PoI 搜索（解决"千灯湖店"等模糊地址）

### 4.3 多维数据看板
- 漏斗分析：page_view → detail_view → apply_click
- 城市健康度：(已填字段数 / 总应填字段数) 评分
- 实时统计图（数字突显 + 暗色卡片）

### 4.4 AI 智能助手「湾湾鲸」

#### 形象与人设
- **粤港澳大湾区中华白海豚**（粉色海豚 SVG）
- 头顶橙色 + 绿色珊瑚芽（呼应大湾区生态）
- 大眼睛 + 白色腮帮 + 微笑嘴
- **3 状态动画**：待机呼吸 / 思考快眨眼 / 说话嘴巴一张一合

#### 核心能力
| 能力 | 实现 |
|------|------|
| 多服务商切换 | 26 个预设（OpenAI 兼容协议 + 腾讯 ADP），数据库 `AIProvider` 表 |
| 流式输出 | SSE 协议，打字机效果 |
| 多会话管理 | `ChatSession` + `ChatMessage`，绑定驿站码隔离 |
| 联网搜索 | DuckDuckGo + 模型原生联网（如混元 Turbo S） |
| 知识库 RAG | jieba 分词 + BM25，Top-5 召回，每文档 max 2 chunk |
| 多模态输入 | 拖拽/粘贴/上传图片，调 GLM-4V-Flash 视觉模型 |
| 引文溯源 | 答案中 `[N]` 角标 + 末尾资料卡片 |

#### Prompt 工程
```
SYSTEM:
你是「湾湾鲸」🐬——粤港澳大湾区的中华白海豚，专门帮应届毕业生使用青年驿站
【人设】粉色形象、自称偶尔说"小鲸"、回答带 🐬💧📍 等小表情
【能力】驿站介绍 / 政策解读 / 申请指导
【规则】优先基于知识库；无数据时明示；用 [N] 引用编号；回答 ≤ 250 字、分点呈现

USER QUESTION + RAG 上下文 + 联网结果 → MODEL → SSE Stream
```

### 4.5 多模态知识库

#### 数据模型
- `KnowledgeDoc`：6 类型（faq / doc / url / note / image / audio）
- `KnowledgeChunk`：切块（500 字 + 80 字重叠），含分词预存
- `CitationLog`：每次问答引文审计

#### 解析管线
| 类型 | 解析方式 |
|------|---------|
| PDF | pypdf 按页 + 切块 |
| Word | python-docx 段落+表格 |
| TXT/MD | 多编码兼容（utf-8/gbk/gb18030） |
| URL | requests + BeautifulSoup 提正文 |
| Image | Pillow 缩放 → base64 → GLM-4V-Flash → 详细描述 |
| Note | 直接切块 |

#### 检索：BM25
- 选 BM25 而非向量检索，原因：
  1. 服务器是腾讯云轻量（无 GPU）
  2. 应届生场景关键词查询为主，BM25 已足够
  3. 零模型加载开销

#### 引文权限分级
| 资源 | 前台 C 端 | 后台 B 端 |
|------|----------|----------|
| 资料标题 + 类型 | ✅ 显示 | ✅ |
| 120 字摘要预览 | ✅ Hover 卡片 | ✅ |
| 完整原文切块 | ❌ | ✅ `/admin/kb/citation/<id>` |
| 原始文件下载 | ❌ | ✅ `/admin/kb/raw/<id>` |
| 网页原始 URL | ❌ | ✅ |
| 图片原图 | ❌ 仅水印缩略图 | ✅ |

### 4.6 双主题
- 暗夜（默认）：深蓝渐变 + 玻璃态 + 冷青强调
- 白天：莫兰迪清亮蓝 + 纯白卡片 + 柔和阴影
- 切换：右上悬浮按钮 + 系统偏好检测 + localStorage 记忆

## 五、数据模型（核心 10 张表）

```
admin                  -- 后台管理员（用户名+bcrypt密码哈希）
station                -- 驿站（含一键申请字段）
access_code            -- 驿站码
access_log             -- 访问/行为埋点
city_policy            -- 政策入口
ai_provider            -- AI 服务商配置
knowledge_entry        -- FAQ
knowledge_doc          -- 多模态知识库主表
knowledge_chunk        -- 切块
citation_log           -- 引文审计
chat_session           -- AI 会话
chat_message           -- 会话消息
```

迁移策略：`db.create_all()` 自动创建新表；旧表新字段用原生 `ALTER TABLE` 自动补齐（兼容 SQLite）。

## 六、关键技术决策

| 问题 | 决策 |
|------|------|
| 是否用向量数据库？ | ❌ 用 BM25。轻量服务器无 GPU；中文场景 BM25 + jieba 已能满足 |
| 是否用 React/Vue？ | ❌ 原生 JS。无构建步骤，部署简单，模板服务端渲染足以支撑 |
| 高德地图选 1.4 还是 2.0？ | ✅ 1.4。2.0 在 WebGL 禁用环境会白屏，1.4 用 Canvas fallback 更稳 |
| AI Key 怎么管？ | ✅ 数据库 `AIProvider` 表 + 后台可视化管理；config 用环境变量兜底 |
| 流式如何避免 ORM session 失效？ | ✅ 路由层提前快照 provider 为 dict + 预构建 messages，再传给生成器 |
| 引文如何脱敏？ | ✅ 前台 hover 仅返回标题/类型/120字预览；原文走 `@login_required` |
| 双主题如何避免闪烁？ | ✅ `<head>` 同步执行预加载脚本 + CSS 变量驱动 + transition .25s |
